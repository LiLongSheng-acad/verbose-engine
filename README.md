这是什么
一个面向安全运营中心的分析能力调度引擎。它在你的检测引擎已经产生告警之后，根据你定义的台账参数，自动决定每类告警该花多少精力去分析——哪些需要人工深度研判，哪些可以自动处理，哪些可以直接忽略。

核心思想：不追求最高精度，而追求刚好够用的安全。

为什么需要它
一个中型企业的安全运营每天可能产生3万到6万条告警，而负责分析的一线人员只有寥寥数人。每条都看，不可能。随机抽检，不可靠。

这套引擎帮你回答一个问题：在有限的人力下，哪些告警值得看，哪些可以放？

答案不凭经验拍脑袋，而基于你台账中的经济损失参数，通过严格的贪心择优算法自动计算最优分析率。

快速上手
1. 准备台账
在 ledger.csv 中填写每类告警的参数：

字段          含义	          示例值

category	告警类别名称	      命令执行

N	        每日平均告警量	      300

t	      单条研判耗时（小时）	0.033

s	      分析师时薪（元）	    34.5

A	        漏报损失（元）	  1000000

alpha	      敏感度系数	        1.0

2. 运行引擎
Bash
pip install pandas
python soc_engine.py
3. 查看结果
引擎会输出每类告警的最优分析率，以及每条告警的裁定结果（人工研判 / 自动分析 / 忽略）。
